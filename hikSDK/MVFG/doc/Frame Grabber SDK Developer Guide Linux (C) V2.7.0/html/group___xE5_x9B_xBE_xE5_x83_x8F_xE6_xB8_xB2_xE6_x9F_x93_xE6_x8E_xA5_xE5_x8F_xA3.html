◆ MV_FG_DisplayOneFrame()

MV_FGCTRL_API int __stdcall MV_FG_DisplayOneFrame ( IN IMAGEHANDLE  hImage,
IN void *  hWnd,
IN MV_FG_DISPLAY_FRAME_INFO *  pstDisplayFrameInfo 
)

Displays one frame of image.

Parameters
hImage[IN] It refers to the image handle. You can use the handle of frame grabbers, cameras, or stream channels.
hWnd[IN] It refers to the window handle.
pstDisplayFrameInfo[IN] It refers to the image information to be displayed.
Returns
Returns MV_FG_SUCCESS for success, and returns corresponding Error Code for failure.
Remarks
When remote display is performed on the Linux platform, OpenGL support for direct rendering mode is required. The OpenGL version should be later than 4.0, and the graphics card driver requires later OpenGL versions.

MV_FGCTRL_API int __stdcall MV_FG_DisplayOneFrameEx ( IN IMAGEHANDLE  hImage,
IN void *  hWnd,
IN MV_FG_DISPLAY_FRAME_INFO_EX *  pstDisplayFrameInfo,
unsigned int  enRenderMode 
)

Displays one frame of image.

Parameters
hImage[IN] It refers to the image handle. You can use the handle of frame grabbers, cameras, or stream channels.
hWnd[IN] It refers to the window handle.
pstDisplayFrameInfo[IN] It refers to the image information to be displayed.
enRenderMode[IN] It refers to the rendering mode, 0: OpenGL.
Returns
Returns MV_FG_SUCCESS for success, and returns corresponding Error Code for failure.
Remarks
When remote display is performed on the Linux platform, OpenGL support for direct rendering mode is required. The OpenGL version should be later than 4.0, and the graphics card driver requires later OpenGL versions. Only the OpenGL mode supports rendering of images over 4GB (requires graphics card support).
