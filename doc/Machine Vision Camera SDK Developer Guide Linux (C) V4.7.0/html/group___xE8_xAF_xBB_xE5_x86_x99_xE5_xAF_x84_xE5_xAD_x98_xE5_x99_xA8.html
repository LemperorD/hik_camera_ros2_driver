Detailed Description

Includes APIs for getting and setting camera register data by register address.

MV_CAMCTRL_API int __stdcall MV_CC_ReadMemory ( IN void *  handle,
IN OUT void *  pBuffer,
IN int64_t  nAddress,
IN int64_t  nLength 
)

Reads data from device register.

Parameters
handle[IN] It refers to the device handle / frame grabber handle.
pBuffer[IN][OUT] It is used as a data buffer, saving memory value that is read. (The memory value of GEV devices, frame grabbers, and devices connected to the frame grabber is stored according to the big-endian mode, and the memory value of other devices is stored according to the little-endian mode.)
nAddress[IN] It refers to the memory address to be read. The address can be obtained from Camera.xml, in a form similar to the XML node value of xxx_RegAddr (Camera.xml will be generated automatically in the current program directory after the device is turned on).
nLength[IN] It refers to the length of the memory to be read.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Remarks
Read the data from certain segment of register.

Writes data to device register.

Parameters
handle[IN] It refers to the device handle / frame grabber handle.
pBuffer[IN][OUT] It refers to the memory value to be written. (The memory value of GEV devices, frame grabbers, and devices connected to the frame grabber is stored according to the big-endian mode, and the memory value of other devices is stored according to the little-endian mode.)
nAddress[IN] It refers to the memory address to be written to. It can be acquired from Camera.xml file, in a form similar to XML node value of xxx_RegAddr.
nLength[IN] It refers to the length of the memory to be written.
Returns
Returns MV_OK for success, and returns corresponding Error Code for failure.
Remarks
Access device for writing data to certain segment of register.
