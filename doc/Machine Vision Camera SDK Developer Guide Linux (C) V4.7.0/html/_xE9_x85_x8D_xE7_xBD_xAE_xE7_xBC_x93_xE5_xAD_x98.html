Buffer Configuration

The SDK supports the following two buffer configuration methods.
  • SDK Allocates Internal Buffer: The SDK internally allocates image buffers for getting image data.
  • Register External Buffer to SDK: You need to allocate multiple buffers and register the allocated buffers to the SDK via APIs, and then the SDK will use the registered buffers to get image data.
Note
You need to actively call the APIs to register external buffers, and the SDK uses its internal buffers by default.

SDK Allocates Internal Buffer

You can call MV_CC_SetImageNodeNum() to set the number of internal image buffer nodes. If the number of buffer nodes is not set, the SDK will use the default number of buffers.
Note
MV_CC_SetImageNodeNum() should be called before MV_CC_StartGrabbing().
The sample code below shows how to get image data using internal buffers.
/* (Optional) Set the number of buffer nodes */
nRet = MV_CC_SetImageNodeNum(handle, 5);
Check(nRet);
/* Start grabbing images */
nRet = MV_CC_StartGrabbing(handle);
Check(nRet);

Register External Buffer to SDK

You can allocate and release external image buffers to and from the SDK.
The basic steps for registering external buffers are as follows:
  1. Call MV_CC_GetPayloadSize() to get sufficient buffer size and correct memory alignment methods.
    Note
    After modifying image-related parameters (such as image width, height, and pixel format), you need to call this API again to reacquire the buffer size.
  2. Call MV_CC_AllocAlignedBuffer() to allocate memory.
  3. Call MV_CC_RegisterBuffer() to register allocated memory as a buffer to the SDK.
  4. Call MV_CC_StartGrabbing() to start grab images.
After image acquisition, you need to cancel the registered buffer as follows:
  1. Call MV_CC_UnRegisterBuffer() to cancel the registered buffer.
  2. Call MV_CC_FreeAlignedBuffer() to release the buffer.
    Note
    • Before calling MV_CC_StartGrabbing(), it is required to register external buffer to the SDK, or internal buffer will be used to get images.
    • It is supported to dynamically register buffers. After calling MV_CC_StartGrabbing() to starting image grabbing, you can call MV_CC_RegisterBuffer() and MV_CC_UnRegisterBuffer() for multiple times.
    • The registered buffer can be used to get images via polling function and via callback function.


// Get the payload information of images in the current configuration
nRet = MV_CC_GetPayloadSize(handle, &nBufLen, &nAlignment);
Check(nRet);
for (int k = 0; k < IMAGE_NODE; k++)
{
// Allocate external aligned memory
pBufferList[k] = MV_CC_AllocAlignedBuffer(nBufLen, nAlignment);
if (NULL == pBufferList[k])
{
nRet = MV_E_RESOURCE;
break;
}
// Register the allocated memory to the SDK
nRet = MV_CC_RegisterBuffer(handle, pBufferList[k], nBufLen, NULL);
if (MV_OK != nRet)
{
break;
}
}
Check(nRet);
/* Start grabbing images */
nRet = MV_CC_StartGrabbing(handle);
Check(nRet);
// Get images
//do something
// Stop grabbing images
nRet = MV_CC_StopGrabbing(handle);
Check(nRet);
for (int k = 0; k < IMAGE_NODE; k++)
{
// Cancel the external buffer registered to the SDK
nRet = MV_CC_UnRegisterBuffer(handle, pBufferList[k]);
if (MV_OK != nRet)
{
break;
}
}
Check(nRet);
// Release user-allocated memory
for (int k = 0; k < IMAGE_NODE; k++)
{
MV_CC_FreeAlignedBuffer(pBufferList[k]);
pBufferList[k] = NULL;
}
