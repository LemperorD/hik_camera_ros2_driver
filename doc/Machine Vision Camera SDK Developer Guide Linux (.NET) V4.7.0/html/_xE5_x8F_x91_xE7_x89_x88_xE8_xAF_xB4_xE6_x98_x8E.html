Release Notes

V4.7.0_November/2025

New Features
  • Adjusted the maximum single write data size of ISerialPort.Write() to 512 bytes.
  • Supported rendering and saving RGB 12 format imagesã€‚
    • Supported rendering RGB 12 format images.
    • Supported calling IImageSaver.SaveImageToFile() [1/2], IImageSaver.SaveImageToFile() [2/2] to save RGB 12 format images.
  • Supported the fast FileAccess feature. If the camera and its firmware support it, this feature can significantly increase the file import and export speed.
Improvements
  • Optimized the API IImageRender.DisplayOneFrame() to support rendering images in PixelType_Gvsp_YUV420SP_NV12 and PixelType_Gvsp_YUV420SP_NV21 formats.
  • Optimized the API IPixelTypeConverter.ConvertPixelType() to support converting pixel format Bayer8 (including BayerRG8, BayerGR8, BayerGB8, and BayerBG8) to RGB8 Planar format.
  • Optimized event related features.
    • Supported registering events for multiple instances of a frame grabber at the same time in a single-process environment. In a process, if you call IEventGrabber.SubscribeEvent() for instance A and instance B respectively to register the same event M, both instance A and instance B will receive event M at the same time.
      Note
      When registering the same event multiple times for the same instance, the previously registered callback function and custom information will be overwritten.
    • SDK supported receiving CoaXPress camera event.
  • Added 0x4008 (ImageBufferOverflow) to StreamExceptionType to indicate that the image data sent by the device has exceeded the capacity of the image buffer.
  • VisionPro plugin supports VP9.23 and VP9.24.
  • Optimized the performance of frame grabber events, reducing CPU usage.
